# 知界 AI 产品方案

## 1. 产品定位

知界 AI 是面向高校课程学习、学科竞赛和自主研究的项目制学习智能体。它解决普通聊天工具中“对话彼此割裂、课程上下文混乱、只给答案不形成学习闭环”的问题。

产品核心不是“上传资料后才能学习”，而是提供两条同等重要的入口：

- **直接对话学习**：学生无需准备文件，可以直接提问、讨论、练习或制定学习计划。
- **结合资料学习**：学生上传课件、教材、题目、实验图、课堂录音等内容，AI 在资料范围内解释、引用和出题。

## 2. 核心差异

### 2.1 多项目学习空间

不同课程的目标和教学方式不同，因此每门课程、竞赛或研究主题建立独立项目。项目分别保存：

- 学习目标与截止时间
- 对话历史
- 用户上传资料
- 知识点与依赖关系
- 掌握度与错题
- 个性化学习路线

### 2.2 从问答升级为学习闭环

完整闭环为：

**目标设定 → 基础诊断 → 路线生成 → 多模态学习 → 练习反馈 → 掌握度更新 → 阶段复盘**

### 2.3 跨学科拓展

跨学科内容不应成为单独堆砌的百科模块，而应由当前知识触发。例如：

- 学习机器学习过拟合时，联系统计学中的偏差—方差权衡；
- 学习电力系统保护时，联系可靠性工程和概率风险；
- 学习英文听力时，联系语音学中的弱读、连读和重音。

系统只在有助于理解或迁移时推荐跨学科概念，避免干扰主线。

## 3. 主要用户流程

### 流程 A：直接开始对话

1. 用户新建“电力系统期末复习”项目；
2. 选择“考前复习”模式；
3. 直接输入“环网供电为什么要设置开环点”；
4. AI 结合用户基础进行解释；
5. 对话结束后抽取知识点并更新掌握度。

### 流程 B：上传资料后学习

1. 用户进入“机器学习竞赛”项目；
2. 上传竞赛任务书和课程讲义；
3. 系统解析文本、图片和表格并建立索引；
4. 用户询问“任务书要求哪些核心功能”；
5. AI 给出带资料来源的回答，并将任务拆成开发路线。

### 流程 C：图片与语音

- 拍摄一道题、原理图或实验现象，AI 先描述识别内容，再讲解；
- 录制课堂片段或口述问题，系统转写后进入正常对话；
- 所有输入都归属当前项目，避免跨课程污染。

## 4. 页面结构

### 4.1 学习总览

显示今日任务、各项目进度、待复习知识点和近期学习统计。

### 4.2 学习对话

核心工作区，包括：

- 项目与模式切换
- 文本、文件、图片、语音输入
- 资料引用
- 快捷追问
- AI 讲解、提问、出题和总结

### 4.3 资料库

管理资料状态：上传中、解析中、可检索、解析失败；支持标签、来源定位和删除。

### 4.4 知识地图

展示知识点、先修关系、掌握度和薄弱点。第一版可用树形列表，后续再升级为图谱可视化。

### 4.5 学习报告

包含学习时长、掌握变化、常见错误、完成任务和下阶段建议。

## 5. 非功能性要求

- 浏览器直接访问，无需安装客户端；
- 移动端可完成基本提问、拍照和录音；
- API Key 必须保存在服务端；
- 用户资料按项目和用户隔离；
- AI 回答明确提示可能出错，资料型回答显示来源；
- 没有文件、没有麦克风权限或模型接口暂时不可用时，核心页面仍可使用并给出明确反馈。
